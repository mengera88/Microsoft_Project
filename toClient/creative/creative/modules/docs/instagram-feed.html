Instagram Feed module

Basic usage

First, add the ajax module <script src="modules/js/ajax.js"></script>, the feedread script <script src="modules/js/ut.feedread.js"></script>, the styles <link rel="stylesheet" type="text/css" href="modules/css/ut.feedread.css">, and pass a DOM element or selector to the instagram_feed() function! Unless you're using a custom sample data file for development, the module will look for creative/data/instagram.json. In the production environment, the data file will be overwritten at regular intervals. (Note that the feed contents will only appear if you're serving the ad via localhost or other means, due to cross-domain origin request limitations.)


    <script src="modules/js/ajax.js"></script>
    <script src="modules/js/ut.feedread.js"></script>
    <link rel="stylesheet" type="text/css" href="modules/css/ut.feedread.css">

    <div id="instagram_feed"></div>

    <script>
        $('#instagram_feed').instagram_feed();
    </script>

Configuration

You can pass a configuration object into the instagram_feed method, and these are the optional properties:


      var defaults = {
        data: 'data/instagram.json', // feed data file path
        count: 10, // how many items to show
        imageSize: 'low_resolution', // (306x306), 'standard_resolution' (~612x612), 'thumbnail' (150x150)
        clickable: true, // user avatars and usernames click through to their profile page
        captions: true, // output markup including the image caption
        stats: true, // output markup displaying like and comment counts
        commentHook: function (commentNum) {}, // should return markup
        likeHook: function (likeNum) {} // should return markup
      }

Events and Links

Clicking on any link generated by the Instagram Feed module is handled by data-uthref, meaning the user will be redirected, and we will track an event based on the id of the element. These ids are dynamically generated based on the unique tweet ID.
