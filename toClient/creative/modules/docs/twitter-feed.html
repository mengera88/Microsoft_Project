Twitter Feed module

Basic usage

First, add the ajax module <script src="modules/js/ajax.js"></script>, the feedread script <script src="modules/js/ut.feedread.js"></script>, the styles <link rel="stylesheet" type="text/css" href="modules/css/ut.feedread.css">, and pass a DOM element or selector to the twitter_feed() function! Unless you're using a custom sample data file for development, the module will look for creative/data/instagram.json. In the production environment, the data file will be overwritten at regular intervals. (Note that the feed contents will only appear if you're serving the ad via localhost or other means, due to cross-domain origin request limitations.)


    <script src="modules/js/ajax.js"></script>
    <script src="modules/js/ut.feedread.js"></script>
    <link rel="stylesheet" type="text/css" href="modules/css/ut.feedread.css">

    <div id="twitter_feed"></div>

    <script>
        $('#twitter_feed').twitter_feed();
    </script>

Configuration

You can pass a configuration object into the twitter_feed method, and these are the optional properties:


      var defaults = {
        data: 'data/twitter.json', // feed data file path
        count: 10, // how many items to show
        clickableTimestamps: true, // timestamps click-through to the tweet's permalink
        avatars: true, // show user avatars
        tweetActions: true // add markup for reply, retweet, and fav buttons
      }

Events and Links

Clicking on any link generated by the Twitter Feed module is handled by data-uthref, meaning the user will be redirected, and we will track an event based on the id of the element. These ids are dynamically generated based on the unique tweet ID.
